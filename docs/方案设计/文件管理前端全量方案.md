# 文件管理前端全量方案

## 1. 概述

在已有 Vite + React + TypeScript 项目基础上，完成一个完整的文件管理 Web 前端应用。项目已初始化（含 antd 6.x、react-router-dom 7.x、zustand 5.x、axios 1.x），类型定义已就绪。

## 2. 架构设计

### 2.1 路由结构

```
/login                          → LoginPage        （公开）
/                               → AuthGuard        （需登录）
  /files                        → MainLayout > FileManager
  /upload                       → MainLayout > UploadPage
  /share/my                     → MainLayout > ShareMyPage
  /share/received               → MainLayout > ShareReceivedPage
  /recycle                      → MainLayout > RecyclePage
  /center                       → MainLayout > UserCenterLayout
    /center/profile             → MainLayout > UserCenterLayout > ProfilePage
    /center/storage             → MainLayout > UserCenterLayout > StoragePage
  /admin                        → MainLayout > AdminLayout
    /admin/users                → MainLayout > AdminLayout > UserManagePage
    /admin/disk                 → MainLayout > AdminLayout > DiskManagePage
    /admin/shares               → MainLayout > AdminLayout > ShareAuditPage
    /admin/sync-logs            → MainLayout > AdminLayout > SyncLogsPage
    /admin/alerts               → MainLayout > AdminLayout > AlertsPage
```

### 2.2 布局层级

```
AuthGuard
  └─ MainLayout (Sider + Header + Content)
       ├─ 标准页面（文件、上传、分享、回收站）
       └─ UserCenterLayout (二级Sider + Content)
       └─ AdminLayout (二级Sider + Content)
```

### 2.3 组件树

```
RouterProvider
└── routes[]
    ├── /login → LoginPage
    └── / → AuthGuard
        └── MainLayout
            ├── Header (Logo + UserDropdown)
            ├── Sider (导航菜单)
            └── Content (Outlet)
                ├── /files → FileManager (Tabs + Tree + Table + ContextMenu)
                ├── /upload → UploadPage (Dragger + Select + ProgressList)
                ├── /share/my → ShareMyPage (Table + Modal)
                ├── /share/received → ShareReceivedPage (Table)
                ├── /recycle → RecyclePage (Table + Popconfirm)
                ├── /center → UserCenterLayout
                │   ├── Sider (个人中心菜单)
                │   └── Content (Outlet)
                │       ├── /center/profile → ProfilePage (Descriptions + Modal)
                │       └── /center/storage → StoragePage (Progress + Table)
                └── /admin → AdminLayout
                    ├── Sider (管理菜单)
                    └── Content (Outlet)
                        ├── /admin/users → UserManagePage (Table + Modal + Search)
                        ├── /admin/disk → DiskManagePage (Card + Form + Button)
                        ├── /admin/shares → ShareAuditPage (Table + Popconfirm)
                        ├── /admin/sync-logs → SyncLogsPage (Table)
                        └── /admin/alerts → AlertsPage (Table + Button)
```

### 2.4 数据流

```
API Layer (src/api/)
  ├── client.ts        — Axios 实例，请求/响应拦截器
  ├── auth.ts          — 认证相关 API
  ├── folder.ts        — 文件夹 API
  ├── file.ts          — 文件/分片上传 API
  ├── share.ts         — 分享 API
  ├── admin.ts         — 管理后台 API
  └── index.ts         — 统一导出

Store Layer (src/stores/)
  ├── useAuthStore     — 认证状态 (token + user)，persist 持久化
  └── useFileStore     — 文件浏览状态 (currentPartition, folderId, fileList, folderTree)

Page/Component Layer
  └── 通过 API 函数调用后端，通过 Store 管理跨组件状态
```

## 5. Store 设计

### 5.1 useAuthStore
```typescript
interface AuthState {
  token: string;
  user: User | null;
  login: (token: string, user: User) => void;
  logout: () => void;
  setUser: (user: User) => void;
}
```
- 使用 `persist` 中间件持久化到 localStorage
- 401 响应时自动调用 `logout()`

### 5.2 useFileStore
```typescript
interface FileState {
  currentPartition: number;        // 0=私有 1=公共 2=分享
  currentFolderId: number | null;  // 当前浏览的文件夹 ID
  fileList: FileInfo[];
  total: number;
  folderTree: Folder[];
  loading: boolean;
  fetchFiles: (params) => Promise<void>;
  fetchTree: () => Promise<void>;
  setPartition: (partition: number) => void;
  setFolderId: (id: number | null) => void;
}
```

## 6. 安全约束

| 约束项 | 实现方式 |
|--------|---------|
| Token 注入 | Axios 请求拦截器读取 useAuthStore.token |
| 401 处理 | Axios 响应拦截器 → logout() → 跳转 /login |
| 危险操作确认 | 删除/取消分享/禁用用户等使用 Popconfirm |
| 权限控制 | role===1 时显示管理入口和 AdminLayout |
| 提交防重复 | Button loading 状态 + disabled |

## 7. 页面交互流程

### 7.1 登录流程
1. 用户填写用户名/密码
2. 调用 `login()` API
3. 成功后保存 token + user 到 useAuthStore
4. 跳转到 `/files`
5. 未登录访问任何页面 → AuthGuard 拦截 → 跳转 `/login`

### 7.2 文件管理流程
1. 顶部 Tabs: 私有/公共/分享
2. 左侧树形文件夹列表
3. 点击文件夹 → 右侧 Table 展示文件
4. 右键菜单：新建文件夹/重命名/移动/删除/分享
5. Table 操作列：下载/预览/删除

### 7.3 上传流程
1. 选择目标文件夹（Select 下拉树）
2. 拖拽或点击选择文件
3. 分片上传：init → chunks循环 → merge
4. 进度条实时显示每文件进度
5. 完成后自动刷新文件列表

### 7.4 分享流程
1. 我的分享：Table 展示 + 创建分享 Modal
2. 创建时选择用户、权限、有效期
3. 可随时取消分享
4. 收到的分享：Table 展示，状态 Tag

### 7.5 管理后台流程
1. 用户管理：Table + 搜索 + 新增/编辑 Modal + 禁用
2. 磁盘管理：信息展示 + 同步配置 + 手动触发
3. 分享审计：全局分享记录 + 强制关闭
4. 同步日志：日志 Table
5. 告警：告警 Table + 批量已读
