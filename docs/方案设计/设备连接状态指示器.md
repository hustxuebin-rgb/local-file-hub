# 设备连接状态指示器

## 1. 需求概述

在三个布局组件中添加当前设备（Server）连接状态指示器，通过调用 `GET /api/lan/server_info` 检测 Server 是否在线。

| 组件 | 位置 | 显示内容 |
|------|------|---------|
| `MainLayout.tsx` | Header 右侧，用户下拉菜单左侧 | Tag + Badge 状态指示 |
| `UserCenterLayout.tsx` | Card title 旁 | Tag + Badge 状态指示 |
| `AdminLayout.tsx` | Card title 旁 | Tag + Badge 状态指示 |

- 在线：绿色 Badge + "已连接 Server"
- 离线：红色 Badge + "Server离线"

## 3. 变更清单

### 3.1 新增 API 函数：`src/api/admin.ts`

```ts
export function getServerInfo(): Promise<ApiResponse<any>> {
  return client.get('/api/lan/server_info').then((res) => res.data);
}
```

- 端点：`GET /api/lan/server_info`
- 返回：`ApiResponse<any>`（成功 = 在线，异常 = 离线）
- 通过已有 `client`（axios 实例）发起请求，自动携带 Token

## 4. 数据流设计

```
组件 mount
  └─ useEffect (空依赖，仅执行一次)
       ├─ getServerInfo() 成功 → serverOnline = true  → 绿色 "已连接 Server"
       └─ getServerInfo() 失败 → serverOnline = false → 红色 "Server离线"
```

- **不设置轮询**：仅在组件挂载时请求一次。原因：避免因周期性请求失败触发响应拦截器的 `message.error` 弹窗刷屏。
- **初始状态**：`serverOnline = false`，渲染完成后根据 API 结果更新。

## 5. 潜在风险与缓解

| 风险 | 缓解措施 |
|------|---------|
| Server 不可达时，axios 响应拦截器会 `message.error` 弹出错误提示 | 仅首次挂载时触发一次，用户体验可接受；后续可考虑在 `client.ts` 中添加 `suppressError` 配置项 |
| 三个组件独立请求 `/api/lan/server_info`，造成三次请求 | 当前场景下三个布局组件很少同时存在（路由互斥），影响极小 |
