# 文件夹管理模块方案设计

**日期**：2026-06-22

## 一、需求概述

实现文件夹管理 RESTful API，支持：
- 列出指定父目录下的文件夹列表
- 创建文件夹（含物理目录创建 + 同级重名校验）
- 重命名文件夹（含同级重名校验 + 级联子文件夹路径更新）
- 移动文件夹（含同级重名校验 + 级联子文件夹路径更新）
- 删除文件夹（含级联删除子文件夹记录）
- 获取当前用户完整文件夹树
- 获取所有公开文件夹列表

## 三、分层设计

### 3.1 Repository 层：`FolderRepo`

```
FolderRepo struct { DB *gorm.DB }
```

| 方法 | 说明 |
|------|------|
| `FindByID(id int64)` | 按主键查单条 |
| `FindByUserAndParent(userID, parentID int64)` | 查某用户某父目录下所有文件夹（sort+create_time 排序） |
| `FindTreeByUser(userID int64)` | 查某用户全部文件夹（parent_id + sort + create_time 排序） |
| `FindPublicFolders()` | 查所有公开文件夹（is_public=1，create_time DESC） |
| `Create(folder *model.Folder)` | 插入 |
| `Update(folder *model.Folder)` | 全字段保存 |
| `Delete(id int64)` | 硬删除 |
| `CheckDuplicate(userID, parentID, name)` | 检查同名（新增时） |
| `CheckDuplicateExclude(userID, parentID, name, excludeID)` | 检查同名（更新时排除自己） |
| `UpdateFullPathByPrefix(oldPrefix, newPrefix)` | 级联更新子文件夹 full_path（REPLACE SQL） |
| `DeleteByUserAndPrefix(userID, pathPrefix)` | 级联删除子文件夹记录 |

### 3.2 Service 层：`StorageService`

```
StorageService struct {
    FolderRepo   *repository.FolderRepo
    FileRepo     *repository.FileRepo   // 预留，当前 nil
    MainDiskPath string
}
```

**错误常量（7个）**：
- `ErrFolderNotFound` — 文件夹不存在
- `ErrFolderNameEmpty` — 文件夹名称为空
- `ErrFolderNameDuplicate` — 同级目录存在同名文件夹
- `ErrParentFolderNotFound` — 父文件夹不存在
- `ErrCannotMoveToSelf` — 不能移动到自身
- `ErrCannotMoveToDescendant` — 不能移动到子文件夹
- `ErrFolderCreateFailed` — 文件夹创建失败

**公开方法**：

| 方法 | 逻辑 |
|------|------|
| `CreateFolder(userID, parentID int64, folderName string, isPublic *int8)` | 1. 校验名称非空；2. 查父文件夹（非0时验证存在且属于当前用户）；3. 重名检查；4. 拼接 fullPath；5. DB 插入；6. 创建物理目录 `os.MkdirAll` |
| `ListFolders(userID, parentID int64)` | 调用 `FolderRepo.FindByUserAndParent` |
| `GetFolderTree(userID int64)` | 调用 `FolderRepo.FindTreeByUser` |
| `RenameFolder(userID, folderID int64, newName string)` | 1. 查原文件夹；2. 校验归属；3. 重名检查（排除自身）；4. 构建新旧 fullPath；5. DB 更新名称+fullPath；6. 级联更新子文件夹 fullPath |
| `MoveFolder(userID, folderID, newParentID int64)` | 1. 查原文件夹；2. 校验归属；3. 禁止移入自身或子文件夹；4. 查新父文件夹存在且归属正确；5. 重名检查；6. 构建新旧 fullPath；7. DB 更新 parentID+fullPath；8. 级联更新子文件夹 fullPath |
| `DeleteFolder(userID, folderID int64)` | 1. 查文件夹；2. 校验归属；3. 级联删除子文件夹 DB 记录；4. 删除当前文件夹记录；5. 删除物理目录 `os.RemoveAll` |
| `GetPublicFolders()` | 调用 `FolderRepo.FindPublicFolders` |

### 3.3 Handler 层：`FolderHandler`

```
FolderHandler struct { StorageService *service.StorageService }
```

**请求体**：
- `CreateFolderReq`：`ParentID int64`、`FolderName string`、`IsPublic *int8`
- `RenameFolderReq`：`FolderID int64`、`NewName string`
- `MoveFolderReq`：`FolderID int64`、`NewParentID int64`

**Handler 方法（7个）**：

| Handler | Method | 路径（相对 /api/folder） |
|---------|--------|--------------------------|
| `ListHandler` | GET | `/list?parentId=0` |
| `CreateHandler` | POST | `/create` |
| `RenameHandler` | POST | `/rename` |
| `MoveHandler` | POST | `/move` |
| `DeleteHandler` | POST | `/delete`（body: `{folderId}`） |
| `TreeHandler` | GET | `/tree` |
| `PublicHandler` | GET | `/public` |

**辅助函数**：
- `getUserID(c *gin.Context) int64`：从 context 提取 `user_id`

### 3.4 Router 层变更

在 `SetupRouter` 中：
1. 依赖初始化区新增：
   ```go
   folderRepo := &repository.FolderRepo{DB: db}
   storageService := &service.StorageService{
       FolderRepo:   folderRepo,
       FileRepo:     nil,
       MainDiskPath: cfg.Storage.MainDiskPath,
   }
   folderHandler := &handler.FolderHandler{StorageService: storageService}
   ```
2. 替换 `folderGroup` 预留空组为 7 条实际路由。
