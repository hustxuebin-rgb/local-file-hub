# 分享管理模块方案设计

**日期**：2026-06-22

## 一、需求概述

实现文件/文件夹分享管理功能，支持：
- 用户将文件或文件夹分享给其他用户
- 查看我分享的和我收到的分享列表
- 查看分享内容详情（含文件/文件夹信息）
- 更新分享权限和有效期
- 取消/关闭分享

同时实现后台清理 Worker：
- 每小时清理过期分享（status=0）
- 每天清理30天以上的回收站文件

## 三、数据模型

使用已有 `model.ShareRecord`，字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| ID | int64 | 主键 |
| ShareType | int8 | 1=文件, 2=文件夹 |
| ResourceID | int64 | 文件/文件夹ID |
| ShareUserID | int64 | 分享发起者 |
| ReceiveUserID | int64 | 接收者 |
| SharePerm | int8 | 1=只读, 2=读写 |
| ExpireType | int8 | 1=永久, 2=7天, 3=30天, 4=自定义 |
| ExpireTime | *time.Time | 具体过期时间 |
| Status | int8 | 1=有效, 0=已取消/已过期 |
| CreateTime | time.Time | 创建时间 |
| UpdateTime | time.Time | 更新时间 |

状态常量：
- `ShareStatusActive = 1` 有效
- `ShareStatusInactive = 0` 已取消/过期

过期类型常量：
- `ExpireTypePermanent = 1` 永久
- `ExpireType7Days = 2` 7天
- `ExpireType30Days = 3` 30天
- `ExpireTypeCustom = 4` 自定义时间

### 业务规则
1. 不能分享给自己
2. 同一资源不能重复分享给同一用户（status=1时）
3. 查看内容时，shareUserID 或 receiveUserID 匹配
4. 关闭分享：发起者关闭（status=0），接收者也关闭（status=0）
5. 更新分享：仅发起者可更新权限和有效期

## 七、Worker 设计：CleanupWorker

依赖：`DB`, `ShareRepo`, `FileRepo`

```go
type CleanupWorker struct {
    DB        *gorm.DB
    ShareRepo *repository.ShareRepo
    FileRepo  *repository.FileRepo
}

func (w *CleanupWorker) Start()  // 启动定时任务
func (w *CleanupWorker) cleanExpiredShares()  // 每小时清理过期分享
func (w *CleanupWorker) cleanRecycleBin()     // 每天清理30天以上回收站文件
```

### 清理逻辑
1. **cleanExpiredShares**：查找 status=1 且 expire_type != 1 且 expire_time < now 的记录，批量 Deactivate
2. **cleanRecycleBin**：查找 is_delete=1 且 delete_time < now-30天 的文件，调用 HardDelete

## 八、技术决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 分层 | repo → service → handler | 遵循项目既有架构 |
| ORM | GORM | 与项目一致 |
| 依赖注入 | 构造函数外注入 | 与项目 router.go 中 DI 模式一致 |
| 定时任务 | time.Ticker goroutine | 简单轻量，无需外部依赖 |
| 分页 | offset/limit | 与现有 file handler 一致 |
| 错误处理 | 显式检查 error | Go 惯用法 |
| 用户信息 | 查 UserRepo | 获取昵称用于展示 |
